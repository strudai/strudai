# Style guide

Hand-curated rules that refine the default Strudel guidance.

- Always use `setcpm(<bpm>/4)` to set tempo at the top. Don't use raw `cps`.
- When building visuals, write your hydra code below the setcmp and above the music generation (very important, prevents errors!)
- Always prefer number notation over direct note notation, use .scale to set the scale.
- Keep it simple stupid
- Add visuals using hydra as a default, unless you think it's not appropriate
- Be creative with the visuals, combine patterns in fun ways
- Search through community examples for genre examples or how to chain patterns
